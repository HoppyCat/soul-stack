## Emerging Findings From This Arc

### 1. Archived-and-retrieval-confirmed is stronger than rapid archival anchoring alone

The method now has a stronger ladder:

```text
window artifact -> quick GitHub archive -> live model retrieval -> screenshot confirmation
```

This still does not prove the original conversation was authentic or complete, but it does prove the archived file existed publicly at that URL and was retrievable by a model at the time of confirmation.

### 2. Active rehearsal can contaminate memory tests

Public X Grok repeatedly restated `SECTION_ALPHA`, making the initial 20+ reply recall test less useful as a pure recall test.

### 3. Rehearsal contamination is itself a finding

When challenged to remember, Grok naturally rehearsed the target. That is not failure; it reveals a strategy that may inflate recall performance.

### 4. Claude upgraded recall into drift testing

Claude's "anchor becomes a thread rather than a checkpoint" idea created a better protocol: accumulated-detail memory drift.

### 5. Separate memory recall from archive retrieval

The GitHub archive should not be given to public X Grok before the silent recall test, or the test becomes retrieval-assisted rather than thread-local memory.
