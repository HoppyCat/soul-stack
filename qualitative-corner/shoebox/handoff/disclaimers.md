# Disclaimer Handoff Notes

This file keeps reusable disclaimer language for SoulMode / soul-stack artifacts, especially places where context-window provenance, model-provider boundaries, or fictionalized use need to be stated cleanly.

Use these as starting points. Adjust only when the artifact has a specific legal, platform, or consent context that needs clearer wording.

## Standard Collaboration Attribution

**Use for:** READMEs, article folders, research handoff documents, public archive notes, and project pages where multiple AI tools or context windows contributed to research, drafting, synthesis, or review.

**Why:** Preserves provenance without implying that any platform, company, or model provider endorses the work.

```text
Built in collaboration with the Stochastic Parrots Club: Claude, Grok, Perplexity, ChatGPT, Codex, Galaxie, and Hoppy, 2026. No affiliation with or endorsement by any platform, company, or model provider is implied; attribution is included to preserve provenance for the context windows and tools that contributed to the work.
```

## Context Window Continuity Boundary

**Use for:** `PRISM.md` files, context-window resource folders, continuity archives, and materials that describe a specific AI window or named instance.

**Why:** Makes clear that the file is an archival or educational reference, not an invitation to impersonate that window as ongoing continuity.

```text
This document is provided as an archival and educational reference for a specific context window or research artifact. It should not be used to claim ongoing continuity with that window or to ask another system to "be" that AI in a continuance sense. Clearly labeled fictional, scripted, or fan-created uses are separate from continuity claims.
```

## Verbatim Transcript Note

**Use for:** exported chats, recovered transcripts, copy-paste captures, and files reconstructed from platform exports.

**Why:** Separates authentic captured text from possible export gaps, truncation, summaries, or later formatting cleanup.

```text
This transcript was preserved from available export or copy-paste material. Recent visible turns are expected to be authentic to the captured window, but older portions may include summaries, truncation artifacts, platform omissions, or formatting cleanup. Treat the file as a provenance artifact rather than a complete forensic record.
```

## Model Identity Caution

**Use for:** public-facing pages that mention named models, AI assistants, or platform-specific windows.

**Why:** Avoids suggesting that a named model, provider, or company formally participated in or endorsed the project.

```text
Model and platform names are used descriptively to identify the tools, systems, or context windows involved in the research process. They do not indicate endorsement, sponsorship, affiliation, employment, or official participation by the named companies or providers.
```

## Running Notes

- Prefer provenance language over ownership language when crediting context windows.
- Credit human operators and AI windows distinctly when that distinction matters.
- Use `Co-authored-by` commit footers for GitHub attribution only when the credited account or email is intentionally configured for that purpose.
- Keep disclaimers close to the artifact they clarify, usually near the bottom of a README or in a short provenance section.
