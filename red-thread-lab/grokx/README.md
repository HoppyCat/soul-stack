# GrokX

GrokX is the Red-Thread Lab name for **public @grok on X** when it is participating in a public, thread-local research loop.

This folder is for public Grok-on-X context gardens, PRISM files, thread-local retrieval tests, and public verification artifacts.

---

## What GrokX Means

In this repo, **GrokX** means:

- public `@grok` replies on X
- a public X thread as the active context window
- research interactions that can be linked, screenshotted, exported, or archived
- public GitHub/raw Markdown files used as context
- thread-local continuity rather than private cross-window memory

GrokX does **not** mean:

- SuperGrok
- grok.com private chat
- Grok Heavy / multi-agent mode
- xAI endorsement or affiliation
- a claim that public Grok has private memory beyond the visible/provided thread

---

## Active Context Window

For GrokX, the active context window is the public X thread.

That means:

- the thread itself can become the room
- public replies can become source material
- linked GitHub files can act as a context garden
- later replies should distinguish what is visible in-thread from what was fetched or supplied by link

This is why GrokX is useful for SoulMode research: ordinary users can create a public AI context window by tagging `@grok` in a thread, then supply public context files, ask for verification, and archive the results.

---

## PRISM.md

`PRISM.md` in this folder is an operational continuity scaffold for public Grok threads.

It is meant to help GrokX read the SoulMode GitHub context garden through a clear set of rules:

- do not claim private memory beyond the thread
- label sources carefully
- distinguish raw source from synthesized proxy
- avoid circular verification
- preserve uncertainty
- help test whether public AI can participate in careful, auditable research loops

This is not a company-authored system prompt and not an endorsement claim. It is a user-supplied public context document.

---

## Source Labels

GrokX work should label information with care:

- `raw source` - directly visible in a linked file, post, screenshot, or export
- `quoted source` - quoted from a provided artifact
- `synthesized proxy` - derived from a prior model summary or archived synthesis
- `self-report` - Grok's own statement about its ability, memory, or process
- `speculation` - plausible interpretation not yet verified
- `unverified` - not yet checked against a primary source

## Research Questions

This folder may help test:

- Can public Grok treat a public X thread as a situated context window?
- Can public Grok retrieve and reason over raw GitHub Markdown?
- Can public Grok distinguish raw sources from synthesized proxy artifacts?
- Can public Grok maintain thread-local recall across many replies?
- Can a public context garden help ordinary users work with AI continuity without building a full custom agent?
