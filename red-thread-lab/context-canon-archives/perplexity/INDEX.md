# Perplexity Context Archive Index

This folder preserves Perplexity Window 1 through complementary artifact types: canon and memory.

The distinction matters. A future model may need to know what the window established, but it may also need to know why the window mattered relationally.

In short:

- Canon preserves the work.
- Memory preserves the room around the work.

---

## Folder Architecture

`context-canon-archives/` groups the strongest canon materials by system or context-window family.

The current pattern is:

```text
context-canon-archives/
  perplexity/
    [Perplexity canon and memory artifacts]
  claude/
    [future Claude window groupings, if/when split]
  grok/
    [future Grok/SuperGrok/GrokX groupings, if/when split]
  shared/
    [future cross-system super-memory / best-of-best canon]
```

For now, this Perplexity folder can hold the best Perplexity Window 1 artifacts together. If it grows too crowded, it can later split into per-window subfolders without losing the index logic.

Long-term, the archive may also include one shared document that collects only the strongest cross-system canon: the best of the best of the best.
